Добработанная программа все также реализует создание системы управления спутниками:
Satellite - абстрактный класс спутников;
ImagingSatellite - наследник Satellite, ДЗЗ - осуществляющий снимки Земли;
CommunicationSatellite - наследник Satellite, Связь - осуществляющий
SatelliteConstellation - спутниковая группировка, она связана со спутниками с помощью агрегации.
Добавлены классы:
SatelliteState - класс управления состоянием активация/деактивация;
EnergySystem - класс, отвечающий за управление энергией.
Новые классы заменяют поля isActive, batteryLevel в родительском классе Satellite.
Эта доработка позволяет реализовать 3/5 принципов SOLID:
- SRP - принцип единственной ответственности: теперь класс Satellite не отвечает за расход энергии и изменение состояния;
- OCP - принцип открытости/закрытости: для изменения логики расхода энергии, нужно изменить только EnergySystem, аналогично с SatelliteState;
- LSP - принцип подстановки Лисков: работа с любыми спутниками в коллекции.

Варианты реализации двух оставшихся принципов:

- ISP - принцип разделения интерфейсов: можно выделить особенности миссий в отдельные интерфейсы (Photographic, Transmitting),
а также сделать интерфейс PerformMission для спутников - имеющих миссию, что позволит сохранить полиморфизм. Интерфейсы позволят
создать многоцелевой спутник, реализующий и фотосъемку и передачу данных, а также спутники без явной миссии;
- DIP - принцип инверсии зависимостей: можно убрать у конструктора спутника создание EnergySystem. Дале создать интерфейс для 
энергетических систем, наследующийся SolarEnergySystem или ChemicalEnergySystem. Тогда в конструктор спутника будем передавать
готовые объекты энергетической системы.

В файле Main.java осуществляется:
- создание 2 спутников связи и 3 спутников ДЗЗ;
- создание и формирование группировки 'RU Basic';
- активация спутников;
- выполнение миссий группировки.
